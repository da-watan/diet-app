# Fit Note

Fit Noteは、スマホで毎日の体重・摂取カロリー・PFC・メモを記録するダイエット補助ツールです。静的Webアプリなので、GitHub Pagesにそのまま公開できます。

## 最新仕様

- 目標体重、現在体重、身長、期限、維持カロリーの設定
- 「今日」「進捗」「履歴」「設定」の4タブ構成
- 体重、体脂肪率、摂取kcal、運動時間、PFC、メモの記録
- 睡眠不足、外食、運動あり、空腹強め、むくみのタグ記録
- 今日の記録済み/未記録状態の表示
- 達成率、目標からの変化量、推奨摂取カロリー目安の表示
- 今日の摂取kcalから残りkcalを表示
- 緑、青、黄、赤による状態表示
- 7日平均、直近変化、記録日数、BMI、残りkg、期限までの日数の表示
- 体脂肪率の7日平均、脂肪量、除脂肪体重の表示
- Canvasによる体重、体脂肪率、摂取kcalの切り替えグラフ
- タイムライン形式の履歴表示、前回比、個別削除、全削除
- サンプルデータ追加
- JSONバックアップの書き出し
- JSONバックアップの復元
- GitHub Pages公開時のPWA対応
- ホーム画面追加用アイコン
- Service Workerによる基本的なオフライン再表示
- `skipWaiting()` と `clients.claim()` による更新反映の改善

## 開き方

ローカルで確認する場合は、`index.html` をブラウザで開きます。

```text
outputs/diet-app/index.html
```

ただし、PWA機能やService Workerは `file://` では有効になりません。ホーム画面追加やオフライン再表示を確認するには、GitHub PagesなどのHTTPS環境で公開してください。

## スマホでアプリ風に使う

GitHub Pagesで公開後、スマホのブラウザからホーム画面に追加できます。

- iPhone: Safariで公開URLを開き、共有ボタンから「ホーム画面に追加」
- Android: Chromeで公開URLを開き、メニューから「ホーム画面に追加」

ホーム画面から起動すると、通常のWebページよりアプリに近い見た目で使えます。

## GitHub Pagesで公開する

### 新しいリポジトリで公開する場合

1. GitHubで新しいリポジトリを作成します。
2. `outputs/diet-app` フォルダ内のファイルを、リポジトリのルートに配置します。
3. すべてのファイルをコミットして `main` ブランチへpushします。
4. GitHubのリポジトリ画面で `Settings` → `Pages` を開きます。
5. `Build and deployment` の `Source` を `Deploy from a branch` にします。
6. `Branch` を `main`、フォルダを `/ (root)` にして保存します。
7. 数分後に表示されるURLへアクセスします。

公開URLは通常、次の形式になります。

```text
https://ユーザー名.github.io/リポジトリ名/
```

### 既存リポジトリで公開する場合

既存リポジトリに置く場合も、GitHub Pagesの公開対象フォルダにこのアプリ一式を配置してください。

公開対象を `/docs` にしたい場合は、ファイル一式を `docs/` 配下に入れて、Pages設定で `main` / `docs` を選びます。

## データ管理

入力した体重、体脂肪率、カロリーなどの個人データは、GitHubには保存されません。各端末・各ブラウザの `localStorage` に保存されます。

- 同じ端末、同じブラウザではページを閉じても記録が残ります。
- 別端末や別ブラウザには自動同期されません。
- ブラウザのサイトデータを削除すると記録も消えます。
- 他人が同じGitHub PagesのURLを開いても、あなたの記録は見えません。
- バックアップしたい場合は右上の書き出しボタンからJSONを保存してください。
- 復元したい場合は「履歴」タブの「復元」からJSONを読み込みます。

## 使い方

1. 「設定」タブで現在体重、目標体重、身長、期限、維持カロリーを入力します。
2. 「目標を保存」を押します。
3. 「今日」タブで体重、体脂肪率、摂取kcal、運動時間、タグ、メモを入力します。
4. 必要に応じて「詳細入力」からPFCを入力します。
5. 「今日を記録」を押します。記録済みの日は「今日の記録を更新」に変わります。
6. 「進捗」タブで体重、体脂肪、目標の指標を確認します。
7. グラフは「体重」「体脂肪率」「摂取kcal」で切り替えできます。
8. 「履歴」タブで過去の記録、前回比、タグ、メモを確認・削除できます。
9. 必要に応じてJSONバックアップを書き出し、別端末や同じ端末へ復元します。

## 更新反映

GitHub Pagesで公開済みのサイトを更新した場合、PWAやService Workerのキャッシュにより旧バージョンが残ることがあります。

現在の `service-worker.js` では、キャッシュ名を更新し、インストール時に `skipWaiting()`、有効化時に `clients.claim()` を実行します。これにより、新しいファイルをpushした後の反映が早くなります。

それでも旧画面が残る場合は、ページ再読み込み、ホーム画面アプリの再起動、またはブラウザのサイトデータ削除を行ってください。


## 注意

このツールは健康管理の補助用です。体調不良、持病、妊娠中、摂食障害の不安がある場合は、医師や専門家に相談してください。
